20n Research Laboratory

Morphology-agnostic robot control via variational free energy minimization. One generative model, any body, no retraining.

Founded January 2026 · Paris · Active
Problem

Transfer across morphologies is unsolved

Standard RL approaches (PPO, SAC, TD3) train policies tied to a specific observation and action space. A policy trained on Walker2d cannot control Ant without retraining from scratch. This means every new robot morphology requires its own training pipeline, reward function, and compute budget.

The core issue: these methods learn input-output mappings, not a model of how bodies work. They have no internal representation of morphology that could transfer.

Approach

Active Inference as a control framework

Instead of learning a policy via reward maximization, 20n implements a generative model that maintains probabilistic beliefs about the world and acts by minimizing variational free energy (Friston, 2010).

The key insight: if the agent has a generative model p(o, s) of how observations o arise from hidden states s, it can infer its state in any body by minimizing prediction error — no reward signal needed.

Variational Free Energy F = DKL[ q(s) || p(s) ] − Eq[ ln p(o | s) ]
Complexity (first term): how far the agent's beliefs q(s) deviate from its prior p(s). Penalizes overfitting.
Accuracy (second term): how well the agent's model predicts actual observations. Rewards good predictions.
Minimizing F means finding the simplest beliefs that still explain what the agent sees.

Action selection uses Expected Free Energy (EFE), which balances exploitation (achieving preferred outcomes) with exploration (reducing uncertainty about the environment):

Expected Free Energy G(π) = − Eq[ ln p(o | π) ] − Eq[ H[ p(s | o, π) ] ]
Pragmatic value (first term): does this policy lead to observations I prefer?
Epistemic value (second term): does this policy reduce my uncertainty about hidden states?
The agent naturally balances goal-directed behavior with information-seeking exploration.
Architecture

Perception-action loop

The architecture implements a hierarchical Predictive Coding network. Each level maintains beliefs (mean μ, precision π) and passes prediction errors upward and predictions downward.

Fig. 1 — Perception-action loop. The generative model maintains beliefs over hidden states, receives prediction errors from sensory encoding, and selects actions by minimizing expected free energy. The morphology adapter maps observation/action dimensions at initialization.

The morphology adapter handles the dimension mismatch between bodies. At initialization, it reads the observation and action space dimensions from the environment and configures the generative model's input/output layers accordingly. No retraining — just remapping.

  • Walker2d: 17-dim observation, 6-dim action (bipedal)
  • Ant: 27-dim observation, 8-dim action (quadruped)
  • Humanoid: 376-dim observation, 17-dim action (full body)
Results

Current benchmarks

Evaluated on MuJoCo environments via Gymnasium. The same architecture instance (identical weights and hyperparameters) is deployed on all three bodies.

Environment Survival (steps) Forward (m) Training required
Walker2d-v4 1000 4.06 0 (runtime)
Ant-v4 1000 — 0 (runtime)
Humanoid-v4 1000 — 0 (runtime)

For context: a trained PPO agent on Walker2d typically achieves 3000+ reward after 2M+ training steps. The 20n agent achieves lower absolute performance but requires zero training steps and transfers across bodies without modification. The two approaches optimize for different things.

MaxSurv (maximum survival duration) is currently the primary metric: the agent maintains balance and forward locomotion for 1000 timesteps across all three morphologies. Forward distance optimization is ongoing work.

Implementation

Technical details

  • Belief updating via gradient descent on free energy (continuous state-space formulation)
  • Hierarchical predictive coding: 3-level hierarchy, precision-weighted prediction errors
  • Action selection: EFE computed over finite planning horizon, softmax policy over discretized action candidates
  • Morphology adapter: linear projection layers initialized from environment spec, no learned weights
  • Inference runs on CPU only (Apple M-series), ~20W power draw, real-time at environment step frequency
Limitations

What doesn't work yet

This is early-stage work. Being honest about limitations:

  • Absolute reward is significantly lower than trained RL agents on the same environments
  • Forward distance on Ant and Humanoid not yet reliably measured (locomotion is stable but not consistently directional)
  • Currently limited to MuJoCo simulation — no physical robot validation
  • Generative model complexity scales poorly with very high-dimensional observation spaces (Humanoid 376-dim is near the limit)
  • No comparison yet with other morphology-agnostic approaches (e.g., Transformer-based universal policies)
  • Hyperparameters (precision, learning rate, planning horizon) were hand-tuned, not systematically optimized
Open questions

Where this goes next

  • Can hierarchical depth improve performance on high-DOF bodies?
  • How does this compare to meta-RL and Transformer-based universal policies on the same benchmarks?
  • Can the generative model learn morphology priors from experience with multiple bodies?
  • Sim-to-real transfer: does the framework hold on physical hardware with noisy sensors?
  • Scaling: what happens with 50+ DOF bodies (e.g., dexterous manipulation)?
